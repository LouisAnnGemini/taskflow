# 任务系统范围扩展讨论：工作任务与生活任务的合并

## 背景
讨论关于是否将日常生活中的小任务和习惯合并到当前以工作任务为主的系统中。

## 利 (Pros)
1. **降低认知负荷**: 统一待办事项，减少切换系统的成本，实现“一个大脑”管理。
2. **整体精力管理**: 在统一视角下评估工作与生活的时间分配，做出更合理的决策。
3. **协同效应**: 利用现有系统的日志和追踪功能管理习惯，无需额外工具。

## 弊 (Cons)
1. **数据污染与报表失效**: 生活任务混入工作任务会导致统计指标（如项目进度、工时）失真，且标签体系可能冲突。
2. **心理上下文混淆**: 工作焦虑可能渗透到生活，生活琐事可能打断深度工作。

## 建议与规避策略
如果未来考虑实行，建议采取以下策略：
1. **架构隔离**: 引入 `Context` 字段（如 Work, Personal, Habit），报表需支持按上下文过滤。
2. **UI/UX 隔离**: 通过不同的标签页或仪表盘分离视图，提供“工作模式”以隐藏非工作任务。

## 结论
- 若系统主要用于项目管理，建议保持纯粹。
- 若系统主要用于个人生产力管理，建议合并，但需优化数据结构以确保不干扰统计。
